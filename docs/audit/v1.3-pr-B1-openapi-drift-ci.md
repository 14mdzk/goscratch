## Design Decisions

### Path normalization

Fiber registers routes with `:param` for named params and `*` for trailing wildcards.
OpenAPI uses `{param}` for named params and `{path}` (or any name) for wildcards.

Normalization strategy (both sides converted to OpenAPI `{param}` style):
- Fiber: `:word` → `{word}`, bare `*` segment → `{wildcard}`
- OpenAPI: already in `{param}` form; trailing `*` path segments get `{wildcard}` substitution

This yields a canonical key of the form `METHOD /path/{param}/{wildcard}` for set-diff.

### Non-API route filter

The following path prefixes are excluded from the Fiber-side route list before diffing:
- `/docs` (Scalar UI + OpenAPI YAML serve)
- `/metrics` (Prometheus scrape endpoint, if ever wired)

HEAD routes emitted by Fiber for every GET route are also excluded (OpenAPI does not enumerate HEAD separately).

### Stub app construction

`cmd/openapi-drift/main.go` creates a bare `fiber.New()` app, then calls `RegisterRoutes` on each module with nil or zero-value dependencies that are safe to pass (no network calls). Modules that require non-nil interfaces receive `nil`-safe no-op stubs defined inline in the binary. The `internal/platform/app` package is NOT imported.
