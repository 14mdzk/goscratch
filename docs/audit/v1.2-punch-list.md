Set both fields in `TestJWTConfig()` to match the runtime defaults. | Surfaced during PR-20 implementation |

## Acceptance Per PR

Same as v1.1 punch-list:

1. Tests added or updated for changed behavior (no test = no merge).
2. Update `docs/features/<feature>.md` if behavior changes.
3. Add a `CHANGELOG.md` entry under `[Unreleased]`.
4. Run `make lint test` clean.
5. Security-affecting PRs (#13, #14, #15, #19): include a "what an operator must change after upgrade" note in the PR body.
