## Design Decisions

- **EWKB format**: SRID-tagged (type flag `0x20000000`), little-endian, 25 bytes: 1 byte order + 4 type + 4 SRID + 8 lon (X) + 8 lat (Y).
- **pgx version**: v5 (`github.com/jackc/pgx/v5 v5.9.2`). No global codec registration needed — plain `sql.Scanner` + `driver.Valuer` suffice for pass-through use in `pool.QueryRow(...).Scan(&point)` and `pool.Exec(..., point)`.
- **Lon = X, Lat = Y**: matches PostGIS / GeoJSON / WKT and C1's `Point{Lon, Lat}` field order.
- **NaN/Inf guard**: `EncodeEWKB` returns `ErrNaNOrInf` before writing any bytes.
- **Decode tolerance**: accepts both endiannesses and WKB with or without SRID flag; rejects non-Point types and SRIDs other than 4326 or 0.
- **Integration test approach**: spins its own `postgis/postgis:18-master` container (no dependency on `testutil.StartPostgres` which only runs the app's three migrations). Creates a `TEMP TABLE geo_test (loc geography(Point,4326))`, inserts via `ST_GeomFromWKB($1, 4326)` and selects back via `ST_AsEWKB(loc::geometry)`.
